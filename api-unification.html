The Challenge

The "N+1" Integration Trap

When we acquired Skillate (TRM), they had zero integrations with the US market ATSs we supported (Workday, Taleo, etc.). Sales was selling "Unified CRM + Automation" packages, but the products were technically siloed.

Building 60 custom integrations for Skillate would have taken years. As you can see from the diagram, a single bi-directional sync requires mapping dozens of touchpoints (Applications, Offers, Hires) across varied latency windows.

Risk Analysis
Estimated Dev Time: 60 Integrations × 3 Months/each = 15 Years (Legacy Roadmap)
The "Trap": A single deep integration (like this Workday flow) is incredibly complex. Replicating this 60x was impossible.
Architecture

The "Universal Adapter" Strategy

Instead of building 60 bridges, I built one. I architected a "Dual-Write" model where Skillate would write its data directly to Sense's Entity Management System (EMS) via our API.

We leveraged the write latency—just a few hundred milliseconds—to ensure that by the time a user refreshed the UI, the data was already present in the unified layer, ready for automation.

The Architecture: Skillate (TRM) writes to `sk_` tables in the central EMS, inheriting all downstream automation capabilities.
Business Impact

Commercial Unblocking

This architecture allowed Skillate to keep their own database for UI performance while relying on Sense for the "Heavy Lifting" of data ingestion.

The Result: We successfully deployed the "Unified CRM" to major enterprise clients, securing $5M+ in ARR from logos that would have otherwise churned.
